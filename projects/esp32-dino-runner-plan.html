1. Project Overview & Objectives

The primary objective of this initiative is to engineer a robust hardware-software integrated system using an ESP32 microcontroller and a monochrome OLED display to recreate the iconic Google Chrome "Dino" game.

Skill Growth

2. Technical Specifications

Hardware Requirements

  • Microcontroller: ESP32 DevKit V1 (30 or 38 pin) for high clock speed and versatile GPIOs.
  • Display: 0.96" or 1.3" Monochrome OLED (SSD1306/SH1106 driver), 128x64 pixels.
  • Input System: Tactile momentary push button for jump/restart interactions.
  • Power: 5V via Micro-USB; optional 3.7V LiPo battery with 3.3V regulation.

Software Architecture

Core Framework: Developed in C++ utilizing the Arduino framework and Wire.h for I2C communication.

Physical & Performance Constraints

  • Refresh Rate Target 30 FPS (loop execution and I2C transfer < 33ms)
  • Memory Strategy Static memory allocation to prevent fragmentation

3. Implementation Phases (Project Timeline)

Phase Focus Key Activities
Week 1 Hardware Integration Connect ESP32 to OLED; "Hello World" graphics tests; verify button wiring and serial feedback.
Week 2 Core Mechanics Develop Dino sprite movements; implement gravity/jumping physics; ground scrolling parallax.
Week 3 Game Logic Program randomized obstacle generation; code collision detection; real-time score tracking.
Week 4 Refinement & UI Design screens; high-score persistence (ESP32 Preferences); finalize enclosure.

6. Contingency Plans (Risk Management)

Potential Risk Impact Mitigation Strategy
I2C Bus Latency Medium Increase I2C clock speed to 400kHz via Wire.begin().
Memory Fragmentation High Avoid String class and new keywords; use static buffers.
Hardware Damage Critical Strict "double-check" pinout protocol before USB connection.
Input Lag Medium Utilize Interrupt Service Routines (ISR) or high-frequency polling.
